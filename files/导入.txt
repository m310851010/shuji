强制校验函数（参返回 QueryResult）, data属性返回该表是否有数据
ValidateTable1File(filePath string) - 校验附表1文件
ValidateTable2File(filePath string) - 校验附表2文件
ValidateTable3File(filePath string) - 校验附表3文件
ValidateAttachment2File(filePath string) - 校验附件2文件

以上函数的实现步骤:
1. 读取Excel文件,失败时插入一条导入记录
2. 解析Excel文件,失败时插入一条导入记录
3. 校验数据,参考文件files/校验提示词_V5_0821.mhtml内的强制校验规则, 失败时插入一条导入记录
4. 校验通过,查询该表是否有数据放进data属性
5. 返回QueryResult


ImportTable1(filePath string) - 导入附表1数据
ImportTable2(filePath string) - 导入附表2数据
ImportTable3(filePath string) - 导入附表3数据
ImportAttachment2(filePath string) - 导入附件2数据

以上导入函数（不做校验, 直接导入, 返回 QueryResult）
实现步骤
1. 读取Excel文件,失败时插入一条导入记录
2. 解析Excel文件,失败时插入一条导入记录
3. 校验数据,失败时插入一条导入记录
4. 清空原表数据
5. 导入数据到数据库, 插入一条导入记录
6. 返回导入结果



### 数据导入的excel数据校验规则, 参考文件
- files/0823数据校验规则-表内校验.mhtml
- files/校验提示词_V6_0823.mhtml


数据excel导入步骤:
1. 前端在表1、表2、表3、附件2 tab页切换, 然后该类型的文件
2. 前端调用对应的`ValidateTableFile`校验该类型的文件是否合法, 使用强制校验规则
3. 校验通过后,再检查该文件是否已上传过, 如果相同的文件已上传过(根据缓存的文件名判断),提示用户"是否覆盖已有数据"
4. 用户选择"是", 前端调用对应的`CopyExcelFile`复制到缓存
6. 后端的导入逻辑为,清空该文件已上传的对应表数据, 解析Excel内容,插入到对应的数据库表,写入数据后调用CopyCacheFile函数把上传的excel文件复制一份
7. 如果需要的话, 使用SM4加密敏感数据


导入规则:
- excel文件内容校验：校验的规则在`files/校验提示词_V5_0821.mhtml`文件内, 数据导入模块只使用强制校验规则,数据数据校验模块的模型校验使用强制校验的第5条、第6条和自动校验规则。
- 写入库时记得查看main.sql文件中的描述, 有的字段需要使用SM4加密存储, 读取时要使用SM4解密
